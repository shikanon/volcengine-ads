# 广告视频打分工作流补全与验收计划

## Summary

基于 `docs/广告视频打分Prompt_内部标准增强版.md`，在当前仓库已有半成品实现的基础上，补全并验收一个可运行的 `video_scoring` 一级工作流。该工作流面向“上传本地广告视频 + 手动选择广告类型（品牌 / 买量 / 创意）→ 调用 `visionVideo` 做结构化评分 → 在任务详情中展示图表、证据、分析和建议”的完整闭环。

本计划不从零设计，而是以当前仓库已出现的 `video_scoring` 相关改动为现状继续推进：先校准规格与共享契约，再检查主进程 pipeline 与前端结果渲染是否一致，最后补齐测试并完成 `typecheck / lint / test / build` 全量验收。

## Current State Analysis

- 需求文档 `docs/广告视频打分Prompt_内部标准增强版.md` 明确提出了三类广告的不同评分维度、合规判断、证据和优化建议，并强调“直接输入完整视频，不要抽帧”。
- `spec.md` 当前已经出现 `video_scoring` 相关内容，说明仓库里已开始把该能力纳入规格，但仍需执行者逐段核对章节编号、输入/输出契约与实际代码是否一致。
- `src/shared/types.ts` 已加入：
  - `TaskType` 的 `video_scoring`
  - `AdVideoScoringCategory`
  - `VIDEO_SCORING_CATEGORY_DEFINITIONS`
  - `VideoScoringInput`
  - `VideoScoringResult`
- `src/shared/workflows.ts` 已加入：
  - `video_scoring` 工作流定义
  - `ingest / score / report_writer` 三个节点
  - 三个评分 Prompt：`video_scoring.brand_score`、`video_scoring.performance_score`、`video_scoring.creative_score`
- `src/main/pipelines/index.ts` 已注册 `videoScoringPipeline`，且 `src/main/pipelines/video-scoring/index.ts` 已存在，说明主进程流程骨架已经落地。
- `src/main/validation.ts` 已有 `video_scoring` 分支，当前实现方向是“仅支持本地视频 + 手动选择类型”。
- `src/renderer/App.tsx`、`src/renderer/pages/Home.tsx`、`src/renderer/pages/Workflows.tsx`、`src/renderer/pages/VideoScoring.tsx` 已接入页面入口。
- `src/renderer/components/TaskTable.tsx` 已出现 `video_scoring` 专属预览、雷达图和条形图渲染代码，说明结果可视化方向已经选定为“在现有任务预览中扩展”，不是新建独立详情页。
- `tests/unit/video-scoring-pipeline.test.ts`、`tests/unit/pipeline-contract.test.ts`、`tests/unit/validation.test.ts`、`tests/e2e/workflows.spec.ts` 已出现 `video_scoring` 相关测试或断言，说明测试骨架也已存在。

## Assumptions & Decisions

- 以当前代码现状为准，`video_scoring` 作为独立一级工作流继续推进，不并入 `copywriting` 或其他已有类型。
- 输入范围按当前实现收敛为“仅本地视频”，不包含抖音链接或历史任务对比。
- 广告类型按当前实现收敛为“用户手动选择”，不做自动分类 Prompt。
- 合规不通过时，任务仍应成功产出 `score.json` 和 `report.md`，允许 `dimensionScores` 为空对象，但必须返回合规问题、分析和建议。
- 结果可视化按当前实现收敛为“复用 `TaskTable` 预览弹层”，不新增独立评分详情页。
- 图表实现按当前代码方向使用 React + SVG/CSS，不新增第三方图表依赖。
